## 有效提示的原则

### 要具体

模糊的请求会导致模糊的结果。提供清晰、详细的说明。

<CodeGroup>

```text 好
创建一个英雄区块，包含：
- 标题："更快地构建网站"
- 副标题："适合所有人的 AI 驱动网站构建器"
- 两个 CTA 按钮："开始使用"（主要）和"观看演示"（次要）
- 背景：从蓝色到紫色的渐变
- 在右侧包含一个模型图像
```

```text 差
制作一个漂亮的英雄区块
```

</CodeGroup>

### 提供上下文

帮助 AI 理解您的目标和受众。

```text
我正在为针对小企业主的 B2B SaaS 产品构建落地页。
该产品帮助管理客户关系。创建一个功能区块，突出显示：
- 轻松的联系人管理
- 自动跟进
- 销售管道跟踪
使用专业、值得信赖的设计。
```

### 使用示例

参考现有网站或设计：

```text
创建一个类似于 Stripe 定价页面的定价区块：
- 三个层级：入门版、专业版、企业版
- 月度和年度切换
- 功能比较列表
- 突出显示专业版层级
```

### 分解复杂请求

对于大型项目，分步工作：

1. "创建包含标题、英雄、功能和页脚的整体页面结构"
2. "现在在功能和页脚之间添加推荐区块"
3. "在页脚之前添加价格表"
4. "最后，在底部添加 FAQ 手风琴"

## 提示模式

### 结构提示

定义布局和结构：

```text
创建具有以下结构的落地页：
1. 导航栏（左侧徽标，右侧菜单项）
2. 英雄区块（全宽，居中内容）
3. 功能网格（3 列）
4. 推荐轮播
5. CTA 区块
6. 页脚（3 列：公司信息、链接、社交）
```

### 设计提示

指定视觉风格：

```text
设计风格：
- 现代简约
- 主色调：#3B82F6（蓝色）
- 次色调：#10B981（绿色）
- 字体：标题使用 Inter，正文使用系统字体
- 按钮和卡片使用圆角
- 微妙的阴影增加深度
- 大量留白
```

### 内容提示

提供或请求内容：

```text
对于英雄区块，使用此内容：
标题："使用 AI 更快发布"
副标题："在几分钟内构建生产就绪的网站，而不是几周"
CTA："免费开始构建"

对于其他区块，生成适当的占位符内容。
```

### 功能提示

描述行为和交互：

```text
添加这些交互：
- 点击导航链接时平滑滚动到区块
- 区块进入视图时淡入动画
- 按钮悬停效果（略微放大）
- 从右侧滑入的移动菜单
- 带有错误消息的表单验证
```

## 常见场景

### 构建落地页

```text
为 [产品名称]（一个 [产品描述]）创建落地页。

目标受众：[描述受众]

所需区块：
1. 带有标题、副标题、CTA 和英雄图像的英雄区块
2. 问题/解决方案区块
3. 功能网格（3 个功能）
4. 社会证明（徽标或推荐）
5. 定价（3 个层级）
6. FAQ
7. 最终 CTA

设计：[现代/专业/有趣/等]
颜色：[主色调和次色调]
```

### 添加功能

```text
在功能区块后添加推荐区块。

要求：
- 在一行中显示 3 个推荐
- 每个推荐包括：引用、作者姓名、作者职位、作者照片
- 添加微妙的背景颜色以区别于其他区块
- 使其响应式（在移动设备上堆叠）

使用这些推荐：
[粘贴推荐内容]
```

### 修复问题

```text
移动菜单无法正常工作。当我点击汉堡图标时，什么也没有发生。
请修复移动导航，使其：
- 从右侧打开滑入菜单
- 显示所有导航链接
- 包含关闭按钮
- 点击菜单外部时关闭
```

### 样式更改

```text
更新配色方案：
- 将主色调从蓝色更改为紫色（#8B5CF6）
- 更新所有按钮以使用新的主色调
- 调整悬停状态为略深
- 更新英雄渐变以使用紫色色调
- 保持其他所有内容不变
```

## 高级技术

### 迭代改进

从广泛开始，然后改进：

1. "创建功能区块"
2. "将功能网格从 3 列改为 4 列"
3. "为每个功能添加图标"
4. "增加功能之间的间距"
5. "添加微妙的悬停效果"

### 条件请求

提供选项：

```text
创建定价区块。如果可能，添加：
- 年度/月度切换
- 功能比较工具提示
- 中间层级上的"最受欢迎"徽章

如果这些太复杂，从简单的 3 层布局开始。
```

### 参考以前的工作

在现有基础上构建：

```text
使用与英雄区块相同的设计风格，创建一个 CTA 区块，包含：
- 引人注目的标题
- 简短描述
- 单个突出按钮
- 与英雄互补的背景颜色
```

## 计划模式

### 何时使用计划模式

在以下情况下使用计划模式：

- 复杂的多页网站
- 具有许多功能的项目
- 当您想在构建之前审查时
- 大范围项目

### 有效的计划提示

```text
我想为一家餐厅构建一个完整的网站。

所需页面：
- 首页（英雄、关于、菜单预览、预订 CTA）
- 菜单（按类别分类的菜单项，包含价格和描述）
- 关于（故事、团队、照片）
- 联系（表单、地图、营业时间、电话）

功能：
- 在线预订系统
- 按类别过滤菜单
- 照片库
- 带验证的联系表单

设计：温暖、诱人、以食物为中心
颜色：大地色调（棕色、橙色、绿色）

请在构建之前创建详细计划。
```

## 讨论模式

### 头脑风暴

使用讨论模式探索想法：

```text
我想为我的摄影业务创建一个网站。
我不确定要包含哪些区块或如何构建它。
您能建议一些选项并帮助我决定吗？
```

### 获取推荐

```text
展示 50 多个项目作品集的最佳方式是什么？
我应该使用网格、轮播还是其他东西？
每种方法的优缺点是什么？
```

### 澄清需求

```text
我需要一个联系表单，但我不确定要包含哪些字段。
B2B 服务业务的标准是什么？我应该询问
公司规模、预算、时间表吗？
```

## 常见错误

### 太模糊

❌ "让它看起来更好"
✅ "增加标题的字体大小，在各区块之间添加更多间距，并使用更浅的背景颜色"

### 一次更改太多

❌ "更改颜色、添加动画、重构布局、添加新区块并更新所有内容"
✅ 一次进行一项更改，测试，然后继续下一项

### 假设上下文

❌ "添加常规区块"
✅ "添加这些区块：功能、推荐、定价、FAQ"

### 不提供内容

❌ "创建关于页面"
✅ "使用此内容创建关于页面：[粘贴内容]"

## 成功技巧

<AccordionGroup>
  <Accordion title="从结构开始">
    在深入细节之前定义整体结构。这有助于 AI 理解大局。
  </Accordion>

  <Accordion title="使用一致的术语">
    在整个对话中坚持使用相同的术语。不要在"英雄区块"和"横幅"之间切换。
  </Accordion>

  <Accordion title="提供视觉参考">
    链接到网站或描述您喜欢的特定设计。"让它像 Apple 的网站"比"让它简约"更清楚。
  </Accordion>

  <Accordion title="逐步测试">
    进行更改，测试它们，然后继续。不要等到最后才预览您的网站。
  </Accordion>

  <Accordion title="对复杂请求要有耐心">
    大型、复杂的网站需要时间。将它们分解为可管理的部分。
  </Accordion>
</AccordionGroup>

## 提示模板

### 落地页

```text
为 [产品/服务名称]（一个 [产品描述]）创建 [行业] 落地页。

目的：[描述目标]
目标受众：[描述受众]

区块：
- 英雄：[描述]
- 功能：[列出功能]
- 社会证明：[推荐/徽标/统计]
- 定价：[层级和价格]
- CTA：[行动号召]

设计：[风格描述]
颜色：[配色方案]
```

### 作品集

```text
为 [职业] 创建作品集网站。

页面：
- 首页：[区块]
- 作品：[项目展示]
- 关于：[简历和经验]
- 联系：[联系表单]

风格：[设计偏好]
展示：[数量] 个项目，采用 [网格/轮播/其他]
```

### 电子商务

```text
为 [产品类别] 创建电子商务网站。

功能：
- 包含 [数量] 个项目的产品目录
- 购物车
- 结账流程
- 产品过滤和搜索
- 产品详情页

设计：[风格]
类别：[列出类别]
```
